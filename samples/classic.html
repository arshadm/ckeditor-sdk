Classic Editor Documentation

Classic editor is an editing solution where the styles of the content are separated from the surrounding page. It is particularly useful when:

  • Admininistration and frontend themes in a CMS are different, in such case inheriting styles from backend theme would make no sense.
  • The edited content is rendered in different locations of a website with possibly different styling, in such case it's more efficient to just focus on creating semantically correct syntax.
  • Providing an editor for variable environments (e.g. a themable CMS), where it's hard to predict how user themes are constructed.

Due to the possible differences between the style of the content within editor and when its rendered on a page, such editors are sometimes called WYSIWYM editors (What You See Is What You Mean)

Classic editor with default styles

Classic editor uses its own, sane default styles for edited content. Note that the font style, size and colors on this sample page and inside CKEditor editing area are different (e.g. the "Apollo 11" header is black, not green).

Classic editor with custom styles

It is possible to specify own precisely selected styles for edited content with config.contentsCss. In the editor below the font size has been increased and the colors of links, headings and striked content have been changed.

By fine-tuning a stylesheet for CKEditor it is in fact possible to make the content look exactly the same both on the page and inside editor.

Classic editor is usually created with CKEditor.replace(). The other methods to create classic editor are:

  • CKEDITOR.appendTo()
  • CKEDITOR.replaceAll()
